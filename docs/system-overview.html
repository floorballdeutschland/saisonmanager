Vollständige Systemübersicht — Rollen, Workflows und Funktionen im Überblick

👥 Benutzerrollen

ADM

Admin

Global oder pro Verband (GameOperation)
  • Vollzugriff auf alle Bereiche
  • Benutzer- und Rechteverwaltung
  • Spielplanung und -verwaltung
  • Spieler- und Lizenzverwaltung
  • Vereine, Teams, Ligen konfigurieren
  • API-Keys und Systemeinstellungen
SBK

Spielberichts-Kommission

Scoped auf Verband (Ost / West / FD)
  • Spieltage und Spiele anlegen / bearbeiten
  • Schiedsrichter (Freitext) in Spielplanung eintragen
  • Alle Spielberichte ihres Verbands verwalten
  • Spielstatus setzen und überwachen
  • Vereins- und Teamdaten lesen
RSK

Referees-Schiedsrichter-Kommission

Scoped auf Verband
  • Verfügbare Schiedsrichter prüfen (Blockierungen)
  • Tentative Ansetzung erstellen
  • Schiedsrichter per Mail benachrichtigen
  • Ansetzung veröffentlichen (E-Mail + Lizenzliste)
  • Sperrtage und Profile der Schiris einsehen
VM

Vereinsmanager

Scoped auf einen oder mehrere Vereine
  • Vereinsdaten und Teams verwalten
  • Spielbericht-Eingabe für Heimspiele
  • Spielbericht für Gastspiele einsehen
  • Spielsekretariats-Links generieren (72h)
  • Schiedsrichter-Lesezugriff (Lizenzstufen)
TM

Teammanager

Scoped auf ein Team
  • Aufstellung und Ereignisse für eigenes Team eintragen
  • Spielbericht für eigene Spiele einsehen
  • Spielsekretariats-Links generieren (72h)
SEK

Spielsekretär

Token-basiert, kein Login erforderlich
  • Spielbericht vollständig ausfüllen via Link
  • Aufstellung, Ereignisse, Schiedsrichter, Checkliste
  • Lizenzlisten beider Teams einsehen
  • Link gültig 72 Stunden
SCH

Schiedsrichter

Eigener Account, Referee-Profil verknüpft
  • Eigenes Profil und Lizenzdaten einsehen
  • Sperrtage selbst pflegen
  • Incident-Reports hochladen (PDF)
  • Lizenzlisten via Ansetzungs-Link abrufen

🗄️ Datenmodell

🏢 GameOperation ──▶ Verband (FD, SBK Ost, SBK West …)
└── 🏆 League ──▶ Liga (Saison, Klasse, Kategorie)
└── 📅 GameDay ──▶ Datum, Arena, Ausrichterverein
└── 🏒 Game ──▶ Spiel (Heim/Gast, Status, JSONB-Spalten)
└── 📋 RefereeAssignment ──▶ RSK-Ansetzung (tentative / published)
👤 Player ──▶ clubs[] (JSONB), licenses[] (JSONB)
🦺 Referee ──▶ Lizenznummer, Lizenzstufe, Club, LV
🏛️ Club ──▶ belongs_to StateAssociation (Landesverband)
⚙️ Setting ──▶ Single-row Systemkonfig (JSONB): Saisons, Lizenzkategorien, Strafen …

Game — wichtige Felder

  • players — Aufstellung Heim/Gast (JSONB)
  • events — Tore & Strafen (JSONB)
  • nominated_referee_string — SBK-Planung (Freitext)
  • nominated_referee_ids — RSK-IDs (Integer[])
  • referee1_string / referee2_string — Spielbericht-Eintrag
  • referee_ids — Lizenznummern (Integer[])
  • game_status — open / started / match_record_closed / finalized
  • checklist_answers — Ja/Nein-Antworten (JSONB)

RefereeAssignment — Felder

  • referee1_id / referee2_id — FK auf Referee
  • status — tentative / published
  • published_at — Zeitstempel Veröffentlichung
  • notified_tentative_at — Zeitstempel Benachrichtigung

Token-basierte Zugriffe (72h)

  • Spielsekretariats-Link (secretary_link)
  • Lizenzlisten-Link (license_list)
  • Incident-Report-Link (referee_report)
